User Guide

Overview

This page offers brief tutorials for usage and configuration of various MS-AutoQC features. Be sure to use the navigation guide on the right (or Ctrl+F) to find what you’re looking for.

Setting up Google Drive sync

To allow access to your workspace from any device, you must first create a project in the Google Cloud Console and generate an OAuth Client ID for your installation of MS-AutoQC.

These actions only need to be performed once. Upon completion, any shared user will be able to use the Client ID and Client Secret to sign in to your workspace.

1. Create a project in the Google Cloud Console

Navigate to the Google Cloud Console. The landing page should look something like this:

Choose Create or select a project, then click New Project.

Name the project MS-AutoQC and click Create.

2. Configure the OAuth consent screen

From the sidebar, navigate to APIs & Services > OAuth consent screen.

For user type, choose Internal and click Create. Fill in the following:

  • App name: MS-AutoQC
  • User support email: [email]
  • Developer contact information: [email]

Once you’re done, click Save and Continue.

On the next screen, select Add or Remove Scopes. Scroll down to the Manually add scopes section and paste in the following:

https://www.googleapis.com/auth/drive
https://www.googleapis.com/auth/gmail.send
https://www.googleapis.com/auth/userinfo.email

Click Add to Table. Then click Update. Then click Save and Continue.

3. Generate the OAuth Client ID

From the sidebar, navigate to APIs & Services > Credentials.

Click the Create Credentials button and select OAuth client ID.

For application type, choose Desktop application. Name the application MS-AutoQC.

Once you’re done, click Create. A dialog window will appear, providing your client ID and client secret.

That’s it – you’re all set! Copy these and save them somewhere accessible. If you’re here from the Getting Started tutorial, click here to head back and use these credentials to set up or sign in to your workspace.

Setting up Slack notifications

To set up Slack notifications, you must create your own Slack bot, generate an OAuth token for it, and give that token to MS-AutoQC in Settings > General.

You only need to do this once! After that, MS-AutoQC will use your token to alert you about QC warnings and QC fails.

1. Create your Slack bot

Navigate to the Slack API console and click the Create New App button.

Choose to create your app From Scratch. Fill in the App Name and your Slack workspace, then click Create App.

2. Generate an OAuth token

From the sidebar, navigate to Features > OAuth & Permissions.

Scroll down to the Scopes section. For Bot Token Scopes, add the following OAuth scopes:

chat:write
chat:write.public
incoming-webhook

Now, scroll back up to the OAuth Tokens for Your Workspace section and click Install to Workspace. Once that’s done, you will have generated your Bot User OAuth Token!

3. Give token to MS-AutoQC

In MS-AutoQC, navigate to Settings > General and paste the token you generated in the Slack API client credentials field. Click Save bot token.

Now, enter the name of the channel you’d like MS-AutoQC to post messages to. Once you’re done, toggle Enable notifications and you’re all set.
